# Documentacao do Projeto

Esta pasta concentra apenas a documentacao essencial para entender e manter o projeto.

## Arquivos Essenciais

Se voce quiser entender a estrutura principal da aplicacao, leia nesta ordem:

1. `00-overview.md`
2. `01-bootstrap-config.md`
3. `02-dto-models.md`
4. `03-auth-and-utils.md`
5. `04-user-flow.md`
6. `05-post-flow.md`
7. `06-comment-flow.md`
8. `07-database-and-migrations.md`
9. `15-arquitetura-atualizada-e-contratos.md`
10. `16-testes-automatizados.md`

## O Que Esta Versionado Aqui

- visao geral da arquitetura
- bootstrap da aplicacao
- DTOs e models
- autenticacao e utilitarios
- fluxo de `user`, `post` e `comment`
- schema e migrations do banco
- estado atual da arquitetura
- estrategia de testes automatizados

## O Que Fica Fora do Repositorio

Materiais de estudo mais extensos, apostilas, guias impressos, roadmaps e planos de carreira podem ser mantidos localmente, mas nao fazem parte da documentacao essencial versionada do projeto.

## Objetivo Desta Pasta

Manter no Git apenas o que e realmente necessario para:

- entender o sistema
- executar manutencao
- estudar a arquitetura principal
- revisar o contrato atual da API
- compreender a estrategia de testes
